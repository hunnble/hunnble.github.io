React-Flux

Flux是脸书自己的前端架构模式，分为三个部分（也有的资料上把Action算作第四个部分），Store、Dispatcher以及View。可以说，但是Flux为React的’V’提供了’M’和’C’的功能。但Flux使用的是单向数据流，所以是一种特别的，不是MVC的MVC。

数据流

Flux使用单向数据流。状态在Store中维护，通过change方法传送数据和触发视图渲染。用户的行为触发Action通过Dispatcher进行分派，从而将数据传送到处理Action的Store中。
下面会分别介绍Flux的几个部分。

  • Dispatcher
  • Action
  • Store
  • View

Dispatcher

Dispatcher调度交互和数据流，负责在Store上注册回调以及管理依赖关系。在Flux模式中它是一个单例。

Action

每个在UI上的行为都会创建一个被发送到Dispatcher的Action，之后由Store调用相应的方法（或者说，行为）。

Store

Store负责封装应用的业务逻辑与应用交互。Store通过注册Action来选择响应哪些Action
Store把内部的数据通过更改时的change事件发送到React的组件当中。
Store有几个特征：

  • Store中包含所有的数据
  • 只有在Store中才发生数据改变
  • Store没有赋值，也就是说在Dispatcher将更改发送给Store，Store更改时新数据传到View中。

View

View来控制视图。组件需要和Store交互，包括添加change事件监听器，监听Store变化以及当组件ComponentWillUnmount时清除组件监听器。
